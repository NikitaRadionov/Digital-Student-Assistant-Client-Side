# UC-12: Промодерировать проект

| Атрибут | Значение |
|---|---|
| **Scope** | Цифровой Студенческий Ассистент |
| **Level** | 🌊 Пользовательская цель (sea-level) |
| **Primary Actor** | ЦППРП (Модератор) |
| **Supporting Actors** | Система (применяет переход статуса, пересчитывает укомплектованность) |
| **Revision** | 1.0 · 2026-05-09 |

---

## Цель в контексте

ЦППРП хочет проверить проект на соответствие стандартам платформы и вынести решение — чтобы качественные проекты стали доступны студентам, а несоответствующие были возвращены на доработку или отклонены.

---

## Стейкхолдеры и интересы

| Стейкхолдер | Интерес |
|---|---|
| ЦППРП | Вынести обоснованное решение по каждому проекту; поддерживать качество каталога |
| Преподаватель | Получить обратную связь при отклонении; увидеть свой проект опубликованным |
| Студент | Видеть в каталоге только проверенные, актуальные проекты |
| Система | Корректно применить переход статуса в зависимости от типа проекта (инициативный / supervisor); обеспечить обоснованный комментарий при отклонении |

---

## Предусловия

1. ЦППРП авторизован

---

## Триггер

ЦППРП открывает очередь на модерацию и выбирает проект для рассмотрения.

---

## Гарантии

### Минимальная гарантия
Статус проекта не изменяется при ошибке валидации комментария или нехватке прав. Существующие заявки на проект не затрагиваются.

### Гарантия успеха (одобрение)
- Проект переходит в статус «Опубликован»; данные модератора, время и комментарий сохранены
- Проект появляется в публичном каталоге
- Укомплектованность проекта пересчитана

### Гарантия успеха (отклонение)
- Supervisor-проект: переходит в статус «Отклонён»; комментарий сохранён
- Initiative-проект: переходит в статус «На доработке»; комментарий сохранён; студент может доработать проект
- Владелец проекта уведомлён об исходе

---

## Основной сценарий успеха (одобрение)

| Шаг | Кто | Действие |
|---|---|---|
| 1 | ЦППРП | Открывает очередь на модерацию |
| 2 | Система | Отображает список проектов, ожидающих рассмотрения, в порядке очерёдности (первые — самые давно поступившие) |
| 3 | ЦППРП | Выбирает проект и изучает его детали |
| 4 | ЦППРП | Принимает решение одобрить проект |
| 5 | Система | Валидирует права модератора → [BR-MOD-01] |
| 6 | Система | Валидирует, что проект ожидает решения по модерации → [BR-MOD-02] |
| 7 | Система | Переводит проект в статус «Опубликован»; сохраняет данные модератора, время и комментарий |
| 8 | Система | Пересчитывает укомплектованность проекта; перенаправляет с уведомлением «Проект опубликован!» |

## Расширения

### 1а. ЦППРП не авторизован

- 1а.1. Система перенаправляет на `/auth/?next=/moderation/`
- 1а.2. ЦППРП проходит авторизацию (включить [UC-02](UC-02-Login.md))
- → Продолжить с шага 1

---

### 5а. Пользователь не является ЦППРП и не is_staff
> [BR-MOD-01]: декоратор `@moderator_required`.

- 5а.1. Система возвращает HTTP 403 Forbidden
- **Сценарий завершается**, статус проекта не изменяется.

---

### 6а. Проект уже не находится на модерации (параллельное изменение)
> [BR-MOD-02]: другой модератор уже вынес решение.

- 6а.1. Система сообщает об ошибке: «Проект уже не ожидает решения по модерации»
- 6а.2. Система отображает сообщение об ошибке
- **Сценарий завершается**, повторного изменения не происходит.

---

### 4б. ЦППРП принимает решение «Отклонить» (supervisor-проект)
> [BR-MOD-04]: `source_type = SUPERVISOR` → терминальный статус.

- 4б.1. ЦППРП вводит комментарий с причиной отклонения
- 4б.2. Система валидирует длину комментария ≥ 100 символов → [BR-MOD-03]
- 4б.3. Система переводит проект в статус «Отклонён»; сохраняет комментарий модератора
- 4б.4. Система перенаправляет ЦППРП в очередь с уведомлением «Проект отклонён»
- **Сценарий завершается**. Проект остаётся в `REJECTED` (терминальный статус).

---

### 4в. ЦППРП принимает решение «Отклонить» (initiative-проект)
> [BR-MOD-05]: `source_type = INITIATIVE` → уникальный возвратный статус.

- 4в.1. ЦППРП вводит комментарий с замечаниями для студента
- 4в.2. Система валидирует длину комментария ≥ 100 символов → [BR-MOD-03]
- 4в.3. Система переводит проект в статус «На доработке»; сохраняет комментарий модератора
- 4в.4. Система перенаправляет ЦППРП в очередь с уведомлением «Проект отклонён»
- Студент может ознакомиться с замечаниями, доработать проект и повторно отправить (UC-10)
- **Сценарий завершается** (для ЦППРП). Цикл для студента продолжается.

---

### 4г. ЦППРП решает «Отклонить», но комментарий короче 100 символов
> [BR-MOD-03]: комментарий обязателен и ≥ 100 символов.

- 4г.1. Система сообщает об ошибке: «Комментарий при отклонении должен содержать не менее 100 символов»
- 4г.2. Система отображает сообщение об ошибке
- → Вернуться к шагу 4 (ЦППРП дополняет комментарий)

---

## Перечень технологических и пользовательских вариаций

| # | Переменная часть | Варианты |
|---|---|---|
| TV-1 | Решение | **(A)** `decision="approve"` → всегда `PUBLISHED`; **(B)** `decision="reject"` → `REJECTED` или `REVISION_REQUESTED` в зависимости от `source_type` |
| TV-2 | source_type при отклонении | **(A)** `SUPERVISOR` → `REJECTED` (terminal); **(B)** `INITIATIVE` → `REVISION_REQUESTED` (returnable) |
| TV-3 | Порядок очереди | FIFO по `updated_at` ascending — старейшие проекты рассматриваются первыми |

## Переходы состояний

```
ON_MODERATION
  │
  ├──[approve]──────────────────────────────► PUBLISHED
  │                                               │
  │                                  (auto: пересчёт укомплектованности)
  │                                               │
  │                                    принятых ≥ team_size?
  │                                          YES → STAFFED
  │                                           NO → (остаётся PUBLISHED)
  │
  └──[reject]──── source_type?
                      │
              SUPERVISOR ──────────────────► REJECTED [terminal]
                      │
              INITIATIVE ──────────────────► REVISION_REQUESTED
                                                      │
                                          (студент дорабатывает)
                                                      │
                                              UC-10 (re-submit)
                                                      │
                                               ON_MODERATION
                                               (цикл повторяется)
```

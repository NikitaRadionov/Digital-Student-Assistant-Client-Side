# Actor-Goal Model

> **System:** Цифровой Студенческий Ассистент  
> **Design Scope:** System  

---

## Границы системы (In / Out)

| In (система делает) | Out (система НЕ делает) |
|---|---|
| Публикация проектов в каталоге | Выдача официальных документов об участии |
| Подача и рассмотрение заявок | Оценивание студентов |
| Модерация проектов (ЦППРП) | Управление учебным планом |
| Рекомендации проектов по интересам | Интеграция с LMS / системой оценок |
| Инициативные проекты студентов | Внешние HR-системы |
| Управление дедлайнами платформы | Финансовые операции |

---

## Акторы

### Первичные акторы

| Актор                        | Роль в системе                                                         | Основная мотивация                                     |
| ---------------------------- | ---------------------------------------------------------------------- | ------------------------------------------------------ |
| **Студент**                  | Ищет и подаёт заявки на проекты; может предлагать инициативные проекты | Попасть в команду интересного проекта                  |
| **Преподаватель (Заказчик)** | Создаёт проекты и отбирает команду                                     | Найти мотивированных студентов для работы              |
| **ЦППРП (Модератор)**        | Модерирует проекты, управляет платформой                               | Обеспечить качество и соответствие проектов стандартам |

### Поддерживающие акторы

| Актор | Роль |
|---|---|
| **Система** | Автоматически пересчитывает укомплектованность (`recalculate_project_staffing`), управляет кэшем рекомендаций, ведёт UniqueConstraint на заявках |

## Actor-Goal List

### Уровни целей (по Cockburn)
- ☁️ — Summary (белый / воздушный змей): несколько сессий, несколько user goals
- 🌊 — User Goal (синий / морской уровень): одна сессия 2–20 мин, coffee-break test
- 🐟 — Subfunction (индиго / рыба): поддерживает user goal; выносится только если переиспользуется

## Backbone (основной поток создания ценности)

```
UC-09 Создать проект
  ↓
UC-10 Отправить на модерацию
  ↓
UC-12 Промодерировать проект → PUBLISHED
  ↓
UC-03 Найти проект (Студент)
  ↓
UC-04 Подать заявку на проект
  ↓
UC-11 Рассмотреть заявки → ACCEPTED
```

Параллельный поток (инициатива студента):
```
UC-06 Предложить инициативный проект → ON_MODERATION
  ↓
UC-12 Промодерировать → PUBLISHED или REVISION_REQUESTED
```

## Приоритет для fully dressed разработки

| Приоритет | UC | Обоснование |
|---|---|---|
| 1 | UC-04 | Центральная цель студента; связывает оба потока |
| 2 | UC-09 | Центральная цель преподавателя; начало backbone |
| 3 | UC-11 | Замыкает основной цикл заявок |
| 4 | UC-12 | Критически блокирует публикацию проектов |
| 5 | UC-06 | Уникальный lifecycle; важен для полноты |
| 6 | UC-03 | Большой scope; рекомендации + поиск |

## Явно исключённые функции

| Возможность | Причина исключения |
|---|---|
| Архивировать проект | Статус `ARCHIVED` есть в модели, но UI/transition не реализованы |
| Закрыть набор вручную | Автоматически системой (`recalculate_project_staffing`); не user goal |
| Удалить проект | Только для DRAFT/REJECTED; слишком узкий scope для отдельного UC |
| Редактировать проект | Subflow UC-09/UC-10; не выделен в отдельный UC |

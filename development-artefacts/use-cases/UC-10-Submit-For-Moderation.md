# UC-10: Отправить проект на модерацию

| Атрибут | Значение |
|---|---|
| **Scope** | Цифровой Студенческий Ассистент |
| **Level** | 🌊 Пользовательская цель (sea-level) |
| **Primary Actor** | Преподаватель (Заказчик) |
| **Supporting Actors** | Система (выполняет переход статуса, сбрасывает данные предыдущей модерации) |
| **Revision** | 1.0 · 2026-05-09 |

---

## Цель в контексте

Преподаватель хочет передать готовый черновик проекта на проверку ЦППРП — чтобы после одобрения проект стал виден студентам в каталоге.

> **Coffee Break Test:** Преподаватель нажал «Отправить на модерацию», закрыл браузер и пошёл на перерыв. Это полноценное завершение сессии — он сделал всё что хотел в рамках этого действия. Ожидание решения ЦППРП — уже другая сессия. Уровень: 🌊.

> **Место в backbone:** UC-09 «Создать проект» → **UC-10** → UC-12 «Промодерировать» → каталог доступен студентам.

---

## Стейкхолдеры и интересы

| Стейкхолдер | Интерес |
|---|---|
| Преподаватель | Передать проект ЦППРП; получить подтверждение что заявка принята в очередь |
| ЦППРП | Получить в очередь только валидные, заполненные проекты |
| Студенты | Видеть в каталоге только прошедшие проверку проекты |
| Система | Корректно выполнить переход статуса; сбросить данные предыдущей модерации при повторной подаче |

---

## Предусловия

1. Преподаватель авторизован
2. Проект существует

---

## Триггер

Преподаватель нажимает кнопку «Отправить на модерацию» на детальной странице проекта.

---

## Гарантии

### Минимальная гарантия
Статус проекта не изменяется при ошибке или отсутствии прав. Данные проекта сохраняются без изменений.

### Гарантия успеха
- Статус проекта изменён на `ON_MODERATION`
- Поля `moderation_comment`, `moderated_by`, `moderated_at` сброшены (очищены от предыдущего цикла)
- Преподаватель перенаправлен на детальную страницу с уведомлением «Проект отправлен на модерацию!»
- Проект появился в очереди ЦППРП (UC-12)

---

## Основной сценарий успеха

| Шаг | Кто | Действие |
|---|---|---|
| 1 | Преподаватель | Открывает детальную страницу проекта |
| 2 | Преподаватель | Инициирует отправку проекта на модерацию |
| 3 | Система | Валидирует, что актор является владельцем проекта → [BR-PRJ-01] |
| 4 | Система | Валидирует, что текущий статус проекта допускает отправку → [BR-PRJ-05] |
| 5 | Система | Переводит проект в статус «На модерации»; сбрасывает данные предыдущей модерации |
| 6 | Система | Перенаправляет преподавателя на детальную страницу проекта с уведомлением «Проект отправлен на модерацию!» |

---

## Расширения

### 2а. Преподаватель не авторизован

- 2а.1. Система перенаправляет на `/auth/?next=/projects/<pk>/submit/`
- 2а.2. Преподаватель проходит авторизацию (включить [UC-02](UC-02-Login.md))
- → Продолжить с шага 1

---

### 3а. Пользователь не является владельцем проекта

- 3а.1. Система отклоняет действие с ошибкой доступа
- **Сценарий завершается**, статус не изменяется.

---

### 4а. Проект находится в недопустимом статусе (например, уже на модерации, опубликован, укомплектован или окончательно отклонён)
> [BR-PRJ-05]: допустимые исходные статусы — только `DRAFT`, `REVISION_REQUESTED`, `SUPERVISOR_REVIEW`.

- 4а.1. Система сообщает об ошибке: «Нельзя отправить проект на модерацию в текущем статусе»
- 4а.2. Преподаватель перенаправлен на детальную страницу проекта
- **Сценарий завершается**, статус не изменяется.

---

### 4б. Проект находится в статусе `REVISION_REQUESTED` (повторная подача после замечаний ЦППРП)
> Этот путь активируется только для initiative-проектов (UC-06) после отклонения в UC-12.

- 4б.1. Статус проекта `REVISION_REQUESTED` входит в допустимое множество → [BR-PRJ-05]
- 4б.2. Система выполняет переход в `ON_MODERATION` (шаги 5–7 без изменений)
- 4б.3. Предыдущий `moderation_comment` (замечания ЦППРП) сбрасывается
- → Продолжить с шага 5 (поведение штатное)

## Переходы состояний

```
DRAFT ──────────────────────────────┐
REVISION_REQUESTED ─────────────────┼──► ON_MODERATION
SUPERVISOR_REVIEW ──────────────────┘        │
                                             │
                               (UC-12 принимает решение)
                                             │
                              ┌──────────────┴──────────────┐
                           approve                        reject
                              │                              │
                           PUBLISHED                  source_type?
                                                 SUPERVISOR → REJECTED [terminal]
                                                 INITIATIVE → REVISION_REQUESTED
                                                              (цикл повторяется)
```

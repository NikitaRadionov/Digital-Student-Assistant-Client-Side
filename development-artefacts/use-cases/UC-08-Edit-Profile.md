# UC-08: Редактировать профиль

| Атрибут | Значение |
|---|---|
| **Scope** | Цифровой Студенческий Ассистент |
| **Level** | 🌊 Пользовательская цель (sea-level) |
| **Primary Actor** | Любой авторизованный пользователь (Студент / Преподаватель / ЦППРП) |
| **Supporting Actors** | Система (сохраняет данные, нормализует теги интересов) |
| **Revision** | 1.0 · 2026-05-09 |

---

## Цель в контексте

Пользователь хочет обновить свои персональные данные — чтобы коллеги видели актуальную информацию, а система формировала релевантные рекомендации (для студентов — на основе `interests`).

---

## Стейкхолдеры и интересы

| Стейкхолдер | Интерес |
|---|---|
| Пользователь | Актуализировать личные данные; задать интересы для улучшения рекомендаций |
| Студент | Указать технологические интересы, чтобы получать релевантные рекомендации проектов (UC-03) |
| Преподаватель | Заполнить профиль для узнаваемости среди студентов и коллег |
| Система | Хранить корректные данные; нормализовывать теги интересов по единому формату |

---

## Предусловия

1. Пользователь авторизован и имеет `UserProfile`

---

## Триггер

Пользователь переходит на страницу `/profile/` и вносит изменения в форму.

---

## Гарантии

### Минимальная гарантия
При ошибке валидации данные пользователя не изменяются. Форма возвращается с сообщениями об ошибках и сохранёнными введёнными значениями.

### Гарантия успеха
- Обновлены `user.first_name`, `user.last_name` (из `full_name`)
- Обновлены `profile.bio`, `profile.interests` (нормализованный список тегов)
- Уведомление «Профиль обновлён»; редирект на `/profile/`

---

## Основной сценарий успеха

| Шаг | Кто | Действие |
|---|---|---|
| 1 | Пользователь | Открывает страницу `/profile/` |
| 2 | Система | Отображает форму с текущими данными: `full_name`, `bio`, `interests` |
| 3 | Пользователь | Изменяет одно или несколько полей и отправляет форму |
| 4 | Система | Валидирует данные → [BR-USR-02], [BR-USR-03] |
| 5 | Система | Сохраняет `user.first_name` / `user.last_name` (split `full_name` по первому пробелу) |
| 6 | Система | Нормализует и сохраняет `profile.bio` и `profile.interests` |
| 7 | Система | Перенаправляет на `/profile/` с уведомлением «Профиль обновлён» |

---

## Расширения

### 1а. Пользователь не авторизован

- 1а.1. Система перенаправляет на страницу входа
- → После авторизации (включить [UC-02](UC-02-Login.md)) вернуться к шагу 1

---

### 4а. `bio` указан, но короче 10 символов
> [BR-USR-03]: если `bio` непустой, он должен содержать минимум 10 символов.

- 4а.1. Система возвращает ошибку: «Слишком коротко — расскажите о себе хотя бы в 10 символах»
- → Вернуться к шагу 3

---

### 4б. `bio` превышает 500 символов

- 4б.1. Система возвращает ошибку: «Описание не может превышать 500 символов»
- → Вернуться к шагу 3

---

### 4в. Один из тегов `interests` короче 2 символов

- 4в.1. Система возвращает ошибку: «Слишком короткий тег: "...". Минимум 2 символа»
- → Вернуться к шагу 3

---

### 4г. Один из тегов `interests` длиннее 50 символов

- 4г.1. Система возвращает ошибку: «Тег слишком длинный: "...". Максимум 50 символов»
- → Вернуться к шагу 3

---

### 4д. Тег содержит недопустимые символы

- 4д.1. Система возвращает ошибку: «Недопустимый тег: "...". Используйте буквы, цифры, дефис, точку, +, #»
- → Вернуться к шагу 3

---

### 4е. Количество тегов `interests` превышает 20
> [BR-USR-02]

- 4е.1. Система возвращает ошибку: «Максимум 20 интересов»
- → Вернуться к шагу 3

---

## Перечень технологических и пользовательских вариаций

| # | Переменная часть | Варианты |
|---|---|---|
| TV-1 | `interests_raw` | Теги через запятую; нормализуются через `normalize_technology_tags` (trim, lowercase); пустая строка → `interests = []` |
| TV-2 | `full_name` | **(A)** «Имя Фамилия» → `first_name="Имя"`, `last_name="Фамилия"`; **(B)** одно слово → `first_name="Слово"`, `last_name=""`; **(C)** пусто → поля не очищаются (split возвращает `[""]`) |
| TV-3 | Роль пользователя | Форма одинакова для всех ролей; контекст страницы отличается (счётчики проектов, заявок, очереди) |

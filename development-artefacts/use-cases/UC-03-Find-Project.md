# UC-03: Найти проект

| Атрибут | Значение |
|---|---|
| **Scope** | Цифровой Студенческий Ассистент |
| **Level** | 🌊 Пользовательская цель (sea-level) |
| **Primary Actor** | Студент |
| **Supporting Actors** | Система (формирует выборку, вызывает движок рекомендаций, отдаёт HTMX-partial) |

---

## Цель в контексте

Студент хочет обнаружить подходящий ему проект — чтобы затем подать заявку (UC-04) или сохранить его для дальнейшего рассмотрения (UC-07).

---

## Стейкхолдеры и интересы

| Стейкхолдер | Интерес |
|---|---|
| Студент | Быстро сузить выборку до релевантных проектов; увидеть персонализированные рекомендации |
| Преподаватель | Его опубликованный проект должен быть виден нужной аудитории |
| ЦППРП | Каталог содержит только прошедшие модерацию проекты |
| Система | Отдавать только `PUBLISHED`-проекты; кэшировать рекомендации; корректно обрабатывать пагинацию |

---

## Предусловия

1. Студент авторизован (для доступа к персональным вкладкам и статусам заявок)
2. В системе существует хотя бы один проект со статусом `PUBLISHED`

> Неавторизованный пользователь обрабатывается в расширении **1а**.

---

## Триггер

Студент переходит на страницу каталога проектов `/projects/`.

---

## Гарантии

### Минимальная гарантия
Студент видит страницу с пустой выборкой и сообщением, если ни один проект не соответствует критериям. Данные студента и проектов не изменяются.

### Гарантия успеха
- Студент видит список `PUBLISHED`-проектов, соответствующих введённым критериям (или все, если фильтры не применены)
- Статусы заявок студента (`user_applications`) корректно отображены на карточках
- Пагинация работает корректно (PAGE\_SIZE = 9)

---

## Основной сценарий успеха

| Шаг | Кто | Действие |
|---|---|---|
| 1 | Студент | Переходит на страницу каталога проектов |
| 2 | Система | Загружает список `PUBLISHED`-проектов, отсортированных по дате (newest first); присоединяет статусы заявок студента для каждой карточки |
| 3 | Студент | Вводит поисковый запрос и/или устанавливает фильтры (теги, размер команды, тип проекта) |
| 4 | Система | Применяет фильтры; возвращает обновлённую выборку без перезагрузки страницы |
| 5 | Студент | Просматривает карточки проектов; открывает детальную страницу заинтересовавшего проекта |
| 6 | Система | Отображает полную информацию о проекте: описание, теги, количество мест, статус заявки студента |
| 7 | Студент | Принимает решение: подать заявку (→ UC-04) или сохранить в закладки (→ UC-07) |

---

## Расширения

### 1а. Студент не авторизован

- 1а.1. Система перенаправляет на `/auth/?next=/projects/`
- 1а.2. Студент проходит авторизацию (включить [UC-02](UC-02-Login.md))
- → Продолжить с шага 1

---

### 3а. Студент переходит на вкладку «Рекомендации»
> Система предлагает персонализированные проекты на основе интересов из профиля.

- 3а.1. Студент открывает вкладку «Рекомендации»
- 3а.2. Система формирует подборку рекомендованных проектов:
  - Если интересы профиля заполнены → движок рекомендаций подбирает проекты; результат кэшируется на 5 минут
  - Если интересы не заполнены → показываются 6 последних опубликованных проектов
- 3а.3. Система отображает проекты с пояснениями, почему каждый рекомендован
- 3а.4. Если интересы не заданы → система предлагает добавить интересы на основе истории активности
- → Продолжить с шага 5

---

### 3б. Студент переходит на вкладку «Закладки»
> Список сохранённых студентом проектов.

- 3б.1. Студент открывает вкладку «Закладки»
- 3б.2. Система загружает сохранённые проекты студента с пагинацией
- 3б.3. Если список закладок пуст → отображается подсказка
- → Продолжить с шага 5

---

### 3в. Студент переходит на вкладку «Мои заявки»
> Сводка всех поданных заявок с счётчиками по статусам.

- 3в.1. Студент открывает вкладку «Мои заявки»
- 3в.2. Система загружает все заявки студента; вычисляет счётчики по статусам (всего / на рассмотрении / принято / отклонено)
- 3в.3. Студент просматривает историю заявок
- → Сценарий завершается (поиск нового проекта не обязателен)

---

### 4а. Поиск не дал результатов (фильтры слишком строгие)

- 4а.1. Система возвращает пустую страницу с флагом `is_filtered = True`
- 4а.2. Система отображает сообщение «По вашему запросу ничего не найдено»
- → Вернуться к шагу 3 (студент смягчает критерии или сбрасывает фильтры)

---

### 4б. Студент фильтрует по типу проекта
> Например, хочет видеть только инициативные или только supervisor-проекты.

- 4б.1. Студент устанавливает фильтр по типу проекта
- 4б.2. Система применяет фильтр к выборке
- → Продолжить с шага 4

---

### 5а. Студент видит на карточке, что уже подавал заявку на этот проект
> `user_applications[project.id]` присутствует в контексте; кнопка меняет вид.

- 5а.1. Система отображает статус существующей заявки вместо кнопки «Откликнуться»: «На рассмотрении» / «Принята» / «Отклонена»
- → Продолжить с шага 5 (студент может выбрать другой проект)

---

### 5б. Проект укомплектован (staffing_state = "full")
> `accepted_participants_count ≥ team_size`; `status = STAFFED`.

- 5б.1. Проект не отображается в каталоге (фильтрация только по `PUBLISHED`)
- → Студент видит только проекты, принимающие заявки

---

## Перечень технологических и пользовательских вариаций

| # | Переменная часть | Варианты |
|---|---|---|
| TV-1 | Обновление выборки | **(A)** HTMX-запрос — сервер возвращает HTML-partial без перезагрузки страницы; **(B)** полный GET-запрос при первом открытии или прямой ссылке |
| TV-2 | Фильтрация | Любая комбинация: текстовый поиск в названии и описании, фильтр по тегам, размеру команды, типу проекта; фильтры аддитивны (AND) |
| TV-3 | Рекомендации | **(A)** движок рекомендаций на основе интересов профиля; **(B)** fallback: 6 последних опубликованных проектов (если интересы не заданы) |
| TV-4 | Кэш рекомендаций | Результат кэшируется на 5 минут с учётом пользователя и текущего набора интересов |
| TV-5 | Пагинация | 9 проектов на страницу; для закладок используется отдельный параметр страницы |
